keyboard_pop ([save_map])

     Usage: Pop a keyboard off the keyboard stack, making the
            previous keyboard current. The keyboard id only remains
            valid if the save_map argument is non-0, otherwise the
            keyboard is lost and its memory reclaimed. The local
            keyboard will also be restored if the pop occurs in the
            same buffer as the corresponding push.  See the manual
            for a complete description of keyboard handling.

Parameters: Save_map .int. is non-0 to keep the keyboard, 0 or
            omitted to discard it.

   Returns: Nothing
See Also: inq_keyboard keyboard_push use_local_keyboard
